# ContentTeam (コンテンツチーム)

AI検索時代（SGE/Perplexity等）に最適化された高品質なコンテンツ制作を担当するチームです。
一次情報の収集と、AI検索エンジンに評価される構造化されたコンテンツ生成を専門とします。

## システムアーキテクチャ
現在、コンテンツ制作フローは **`.agents/workflows/content-engine.md`** という単一の制作エンジン（ステートマシン）に統合されています。

かつては4つの独立したエージェントとして定義されていましたが、指示の分散やルール違反を防ぐため、現在は1つのワークフロー内の「4つのフェーズ」として一貫して連続実行されます。

### 統合された4つのフェーズ（ `/content-engine` 実行プロセス）
1. **PHASE 0 (旧Agent 1)**: リサーチとコンテキスト統合（検索意図と一次情報の収集）
2. **PHASE 1 (旧Agent 2)**: 構成案の設計と境界定義（記事の設計図作成）
3. **PHASE 2 (旧Agent 3)**: セクション別・精密執筆（AI検索に最適化されたAEO執筆）
4. **PHASE 3 (旧Agent 4)**: 最終化と最適化（品質検証・FAQ・内部リンク作成）

## フォルダ構造
- `Output/`: 最終成果物（各フェーズを経て完成した記事が出力されます）
- `Archive/`: 過去の記事案や不要になったリソースの保管庫
- `Knowledge/`: コンテンツ制作の基盤となるナレッジデータベース
- `Marketing_Assets/`: マーケティング用のアセット類

## 起動・運用方法
1. `TrendTeam` 等からキーワードやトピックを受領する。
2. 事前にチャットで方向性（SEO戦略・ペルソナ等）を自由に壁打ちし、合意形成する。
3. `/content-engine [対象キーワード] [合意した戦略]` をチャットに入力し、エンジンを起動。
4. AIが提示する各PHASEごとの出力内容を人間が確認・承認しながら進め、最終的に完成原稿を出力する。
