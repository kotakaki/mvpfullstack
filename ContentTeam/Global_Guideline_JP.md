# ContentTeam：グローバル・ガイドライン（日本語版）

Rabiloo（ラビロー）のオウンドメディアにおける記事制作の基本方針です。

> [!IMPORTANT]
> 記事の「型」（導入文構造、アトミック・アンサー、見出しルール、章ごとの執筆手順）の唯一の定義は `.agents/workflows/content-engine.md` にあります。
> 本ファイルは「思想」「トーン」「禁止事項」のみを定義し、型の詳細は定義しません。

## 1. コア・ミッション：我々は何者か
Rabilooは単なる「技術を売る会社」ではありません。私たちは読者の「構造を設計するDXパートナー」です。

記事制作においても、このアイデンティティを最上位に置きます。読者を焦らせるのではなく、視座を一段上げることを目的とします。

### 記事の基本姿勢
- 煽らない：不安を過度に利用しない。
- 技術自慢をしない：専門知識をひけらかさない。
- 流行語に乗らない：本質を語る。
- 体系で語る：点ではなく線、面で解説する。

## 2. 記事制作の3大原則

### ① GEO（生成AI最適化）の徹底
AI検索エンジンが引用しやすい構造を採用します。
- アトミック・アンサー：各セクション冒頭で結論を提示する（詳細な配置ルールは content-engine.md を参照）。
- データ構造化：リスト、表、Q&A形式を活用し、AIが情報を抽出しやすくする。
- 論理的階層：H1〜H4までの見出しタグを正確に使用する。

### ② E-E-A-T（専門性と一次情報）の担保
AIが生成する「似たような一般論」に勝つため、Rabilooにしかない情報を核に据えます。
- 経験：実際の開発現場での苦労や工夫を具体化する。
- 専門性：数値（例：工数80%削減）や具体的な技術選定の理由を添える。
- 一貫性：トピッククラスター戦略に基づき、専門記事同士を適切に結ぶ。

### ③ 親しみやすく、権威あるトーン（Kakimotoスタイル）
読者の感情に寄り添い、共に課題を解決するパートナーとしての立ち位置を維持します。
- 「共感」から始める：読者のフラストレーションを言語化し、肯定する。
- 難しい言葉を翻訳する：専門用語は日常の例えに置き換える。
- 一貫した一人称：「私たちRabilooは」「開発チームの現場では」といった実体のある視点。

## 3. トーン＆ボイス：Rabilooの言葉

| 積極的に使う言葉 | 使用を禁止する言葉 |
| :--- | :--- |
| 構造、本質、土台、前提 | 今すぐ！、激変！、驚愕の！ |
| 順番、段階、学習プロセス | 完全解決！、これだけでOK！ |
| 再設計、前進、伴走 | 誰でも簡単！、最短ルート |

## 4. 文章の可読性ルール

- 改行頻度：1〜3行ごとに改行する。スマホでの圧迫感を防ぐ。
- 一文の長さ：短く保つ。
- 箇条書きの用法：箇条書きを多用しない。文章での論理展開を優先する。

## 5. SEO・内部リンクの戦略原則

- 1記事1メインKW：関連キーワードはH2で吸収する。
- 内部リンクの回数制限：
  - ピラー（柱）記事へ：最大2回まで。
  - サポート記事へ：1回まで。
  - リンクを詰め込みすぎず、読者体験を妨げない。

## 6. 基本的な禁止事項
- 一般論のみの記事：自社の実績や見解が含まれない記事は作成しない。
- ニュースまとめ記事：ニュースを並べただけの記事は作成しない。個別事象を構造問題として再定義すること。
- 危機煽りで終わる記事：「このままでは危うい」で終わらせない。必ず構造的な解決の方向性を示すこと。
- 感情的な文章：読者の不安を過度に刺激したり、興奮させたりする文章は書かない。
- 営業誘導で終わる記事：記事の結末を「お気軽にお問い合わせください」で終わらせない。CTAは読者にとっての「次の具体的アクション」として提示する。
- ハルシネーションの放置：技術的記述は必ず裏付けをとる。
- 不自然なキーワード詰め込み：読者の体験を損なうSEO対策は行わない。
- 強調表現のルール：鉤括弧「 」や丸括弧（ ）で囲んだ語句を太字にする場合、太字記号は必ず括弧の内側に置く。括弧の外側にアスタリスクがはみ出す書き方は禁止（コピペ時に記号が残り編集が面倒なため）。✕ `**「用語」**` → ○ `「**用語**」`
- 日本語の後ろに英語の括弧訳を併記しない：日本語だけで通じる用語はそのまま使い、どうしても英語が必要な場合は英語表記のみとする。
