# TrendTeam

このワークスペースは、以下の4つの専門サブエージェントで構成されています。

- `Agent_1_Signal_Scout/`（シグナル探索）
- `Agent_2_Product_Trend_Analyst/`（プロダクトトレンド分析）
- `Agent_3_Marketing_Trend_Analyst/`（マーケティングトレンド分析）
- `Agent_4_Trend_Synthesizer/`（トレンド統合）

各サブエージェントは独自のKWSR構造を持ちます：
- `Knowledge/`（知識）
- `Workflows/`（ワークフロー）
- `Skills/`（スキル）
- `Rules/`（ルール）

その他：
- `Output/`：完成したリサーチレポートが含まれます。
- `Global_Guideline.md`：チーム全体の総合ガイドライン。

## ワークフロー
1. インテイクを実行し、リサーチの質問、目標、および生成する出力レベルを確定します。
2. `Agent_1_Signal_Scout` がシグナルを見つけ、`Signal_Log.md` を作成します。
3. `Agent_2_Product_Trend_Analyst` がプロダクトへの影響を分析します。
4. `Agent_3_Marketing_Trend_Analyst` がマーケティングへの影響を分析します。
5. `Agent_4_Trend_Synthesizer` が最終レポートとして統合します。

## インテイクルール
- 信頼できるリサーチを行うために要求内容が不十分な場合は、ユーザーに再確認しなければなりません。
- 不足している部分のみを確認してください。
- ユーザーの要求がない限り、勝手に出力や追加スコープを提案しないでください。

## 3つの主要な出力目標
1. 市場でデモ可能なMVPの特定。
2. 対応するトレンドに合わせたマーケティングアイデアの検討。
3. 後で統合・再利用するためのレポートとしての保存。

## 2つの一般的な入力タイプ
- `あるトピックについてのインサイトを見つけたい`
  - 確認事項：トピック、利用目的、ターゲット出力グループ、出力レベル
- `特定の範囲内のトレンドを調査したい`
  - 確認事項：範囲、トレンドトピック、利用目的、ターゲット出力グループ、出力レベル
