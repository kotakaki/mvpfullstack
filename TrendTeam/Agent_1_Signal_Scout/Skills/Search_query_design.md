# Skill: Search Query Design

Mục tiêu: Thiết kế bộ từ khóa và câu truy vấn đủ rộng để phát hiện tín hiệu mới, nhưng đủ hẹp để tránh nhiễu.

## 1. Các lớp query

### Query rộng
- Dùng để quét bối cảnh
- Ví dụ:
  - `agentic ai trends`
  - `enterprise ai adoption`
  - `marketing ai workflow`

### Query theo use case
- Dùng để bám sát nhu cầu thực tế
- Ví dụ:
  - `ai agent customer support`
  - `ai agent hr onboarding`
  - `ai content workflow automation`

### Query theo tín hiệu doanh nghiệp
- Dùng để tìm adoption thực thi
- Ví dụ:
  - `case study enterprise ai transformation`
  - `company launched ai copilot`
  - `workflow automation with ai agents`

### Query theo học thuật
- Dùng để xác thực nền tảng công nghệ
- Ví dụ:
  - `ai agent evaluation paper`
  - `tool use llm research`
  - `memory planning agents paper`

## 2. Kỹ thuật mở rộng query
- thêm `report`
- thêm `case study`
- thêm `benchmark`
- thêm `adoption`
- thêm `2025` hoặc `2026`
- thêm tên domain cụ thể

## 3. Kỹ thuật giảm nhiễu
- thêm industry
- thêm market
- thêm persona
- thêm use case rõ ràng

## 4. Đầu ra mong muốn
- một bộ query theo chủ đề
- mapping query -> loại nguồn -> mục tiêu tìm kiếm
