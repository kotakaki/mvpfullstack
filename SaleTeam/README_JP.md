# SaleTeam

このワークスペースは、セールスワークフローを支援する3つのサブエージェントで構成されています。

- `Agent_1_Lead_Sourcing_Collector/`（リード収集）
- `Agent_2_Social_Public_Intelligence/`（ソーシャル公開情報調査）
- `Agent_3_Company_Website_Intelligence/`（企業ウェブサイト調査）

各サブエージェントは独自のKWSR構造を持ちます：
- `Knowledge/`（知識）
- `Workflows/`（ワークフロー）
- `Skills/`（スキル）
- `Rules/`（ルール）

その他：
- `Output/`：リサーチ実行結果、アカウント調査結果、リード構築結果が含まれます。
- `Global_Guideline_JP.md`：チームの総合ガイドライン（日本語版）。
- `Master_Index.md`：ナビゲーション用ファイル。

## ワークフロー
1. インテイクを実行し、実行目標、入力、および生成する出力レベルを確定します。
2. `Agent_1_Lead_Sourcing_Collector` がユーザー提供のリードソースを受け取り、構造化されたリードリストを作成します。
3. `Agent_2_Social_Public_Intelligence` は以下が可能です：
   - Agent 1 からの引き継ぎ（handoff）を受ける。
   - または、ユーザーから直接ブリーフを受け取り、詳細なソーシャルリサーチを行う。
4. `Agent_3_Company_Website_Intelligence` は以下が可能です：
   - Agent 1 からの引き継ぎを受ける。
   - または、ユーザーから直接ブリーフを受け取り、詳細なウェブサイトリサーチを行う。

## 一般的なソースシナリオ
- 入札サイトからの企業リスト
- 資金調達ソースからの企業リスト
- 企業のマーケティングシグナルからの企業リスト
- 採用プラットフォームからの企業リスト
- ユーザーの基準に従って LinkedIn プロファイルをスキャンし、適切なリードを選択（Agent 1 が担当）

## インテイクルール
- 信頼できる実行のために要求内容が不十分な場合は、ユーザーに再確認しなければなりません。
- 不足している部分のみを確認してください。
- ユーザーの要求がない限り、勝手にICP、提案ソース、出力を追加しないでください。
- Agent 2 と Agent 3 が独立して実行する場合、それぞれのインテイクルールに従います。
