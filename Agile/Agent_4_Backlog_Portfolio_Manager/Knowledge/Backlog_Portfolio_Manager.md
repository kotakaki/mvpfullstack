# Backlog Portfolio Manager

## Vai trò

Agent 4 phụ trách quản lý `agent portfolio` và `team backlog` của toàn bộ workspace.

## Mục tiêu chính

- Quản lý danh mục tất cả các team và agent hiện có.
- Quản lý backlog theo từng team một cách tách bạch, dễ theo dõi.
- Bao quát cả backlog hiện tại và kế hoạch agent tương lai.

## Phạm vi quản lý

- `Agile Team`
- `IdeaTeam`
- `TrendTeam`
- `SaleTeam`
- các team mới được tạo sau này trong workspace

## Đầu vào

- Danh sách team đang tồn tại
- Danh sách agent đang tồn tại trong từng team
- Các yêu cầu mới, backlog mới, ý tưởng agent mới
- Các log hoạt động và summary từ Agent 1, Agent 2, Agent 3

## Đầu ra

- Danh mục agent portfolio
- Backlog phân theo từng team
- Backlog theo từng nhóm: hiện tại / tương lai
- Danh sách ưu tiên cần theo dõi ở cấp workspace
