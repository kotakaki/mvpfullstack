# Activity Signal Collection

## Mục tiêu

Thu thập các tín hiệu cho thấy workspace đã có thay đổi đáng kể.

## Tín hiệu cần thu thập

- File mới được tạo
- File quan trọng được sửa
- Thư mục `Output` có run mới
- Git status có thay đổi
- Git log có commit mới

## Cách thu thập

- Quét thư mục cùng cấp với `Agile`
- Đối chiếu trước / sau khi thực hiện một yêu cầu lớn
- Kiểm tra git status và git log

## Nguyên tắc

- Chỉ ghi nhận sự kiện có giá trị theo dõi
- Không cần log từng thao tác nhỏ nếu không tạo ra kết quả hữu hình
