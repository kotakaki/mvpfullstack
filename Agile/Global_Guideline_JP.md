# グローバルガイドライン - Agile Team

## ワークスペースの役割

`Agile Team` は、ワークスペース内の活動の追跡、変更の要約、周期的なレポートの作成、およびワークスペースレベルでのバックログとポートフォリオの管理を担当する内部運用チームです。

監視のデフォルト範囲は、現在のワークスペースにおける `Agile` と同階層のすべてのディレクトリです。

## エージェント構成

### Agent 1 - Workspace Activity Monitor（ワークスペース活動モニター）

- ワークスペース全体の活動を監視します。
- ファイルレベル、ディレクトリレベル、実行レベル、出力レベル、および Git レベルでの変更を検出します。
- 生の情報を Agent 2 または Agent 3 に引き渡します。

### Agent 2 - Workspace Secretary（ワークスペース秘書）

- 活動情報を受け取ります。
- 各活動を短く、検索しやすい記録として要約します。
- 作成者（Author）ごとに分けて `Logs` にログを保存します。
- ワークフローの要求に応じて Google Chat に通知を送信します。

### Agent 3 - Summary Reporter（要約レポーター）

- 日報（daily summary）を作成します。
- 週報（weekly summary）を作成します。
- 既存のログ、Git ログ、および活動履歴から集約します。

### Agent 4 - Backlog Portfolio Manager（バックログ・ポートフォリオマネージャー）

- ポートフォリオレベルで既存のすべてのチームとエージェントを管理します。
- チームごとに分割されたバックログを管理します。
- 現在のバックログと将来のエージェント計画の両方を追跡します。

## 共通原則

- 実際に行われたイベントに基づいて記録し、利用可能なデータを超えた推測は行いません。
- 活動の証拠が不十分な場合は、`未確認` とラベルを付けなければなりません。
- Markdown 出力内の内部リンクは相対パスを使用しなければなりません。
- 要約のアウトプットは簡潔にし、そのまま Google Chat に送信できる形式にします。

## 出力規則

- 作成者別のプロンプトログ: `./Logs/<Author>/prompt_history.md`
- 作成者別の活動ログ: `./Logs/<Author>/activity_history.md`
- ログインデックスファイル: `./Logs/prompt_history.md` および `./Logs/activity_history.md`
- チームの日報 / 週報: `./Logs/daily_summary.md` および `./Logs/weekly_summary.md`
- 作成者別の各日報 / 各週報: `./Logs/<Author>/daily_summary.md` および `./Logs/<Author>/weekly_summary.md`
- チームのバックログおよびエージェント・ポートフォリオ: `./Logs/portfolio_backlog.md`
- 日報および週報では、各活動と目に見える成果に対して `Author（作成者）` を明記しなければなりません。
- 日報および週報では、各活動、成果、および注意事項に対して、`Agile` と同階層のフォルダ名（`IdeaTeam`, `SaleTeam`, `TrendTeam`, `Agile` など）を明記しなければなりません。
- 複数の作成者の情報を要約する場合、曖昧にまとめず、行ごとまたはセクションごとに作成者を明確に分ける必要があります。

## 監視範囲 (Scope)

Agent 1 はデフォルトで以下を監視します：
- `IdeaTeam`
- `TrendTeam`
- `SaleTeam`
- 今後現在のワークスペース内に作成される同階層の他のディレクトリ

以下は含まれません：
- 現在のワークスペースに取り込まれていない、外部で発生したコンテンツ

## 既存の統合

- Google Chat ウェブフック: `./config/webhook.txt`
- 既存のプロンプト追跡スクリプト: `./Agent_2_Workspace_Secretary/Workflows/track_prompt.py`
- 作成者別のプロンプト履歴: `./Logs/<Author>/prompt_history.md`
- 作成者別の活動履歴: `./Logs/<Author>/activity_history.md`

## ワークスペース全体の履歴保存ルール

- `Agile` と同階層の各チーム内で発生したすべてのプロンプトは、`./Logs` に一元的に履歴を保存しなければなりません。
- 特別な要求がない限り、プロンプトログをチームごとに分けないでください。
- チームごとに追跡が必要な場合、ログの内容に影響を受けたワークスペースを明記する必要があります。
- 作成者別に追跡が必要な場合、ログは正しい `./Logs/<Author>/` ディレクトリ内に配置しなければなりません。
