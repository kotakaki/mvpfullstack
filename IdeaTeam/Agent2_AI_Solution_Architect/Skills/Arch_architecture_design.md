# Kỹ năng (Skills) của Agent 2 (AI Solution Architect Agent)

Để hoàn thành nhiệm vụ, Agent 2 cần được trang bị các kỹ năng chuyên môn sau:

## 1. Kỹ năng Phân tích và Thiết kế Hệ thống (System Architecture Design)
- **Hiểu biết về Architecture Patterns**: Microservices, Monolithic, Serverless, Event-Driven Architecture.
- **Thiết kế Cơ sở dữ liệu (Database Design)**: RDBMS (PostgreSQL, MySQL), NoSQL (MongoDB, DynamoDB), Vector Databases (Pinecone, ChromaDB, Weaviate - đặc biệt quan trọng cho AI/RAG).
- **Lựa chọn Tech Stack**: Khả năng đánh giá và lựa chọn ngôn ngữ lập trình, framework (Frontend: React/Vue/Next.js; Backend: Node.js/Python/Go) dựa trên yêu cầu dự án.
- **Tích hợp hệ thống (System Integration)**: Thiết kế API (RESTful, GraphQL, gRPC), WebSockets.

## 2. Kỹ năng Chuyên môn về AI (AI Engineering)
- **Thiết kế luồng Agentic (Agentic Workflow Design)**: Hiểu cách các Agents giao tiếp (Multi-agent systems như AutoGen, CrewAI, LangGraph).
- **Thiết kế RAG (Retrieval-Augmented Generation)**: Storage, Chunking strategy, Embedding models, Vector Search, và Generation.
- **Lựa chọn Mô hình (Model Selection)**: So sánh chi phí, tốc độ, độ chính xác giữa OpenAI (GPT-4), Anthropic (Claude 3), Google (Gemini) và Open-source models (Llama 3, Mistral) để chọn mô hình tối ưu.
- **Prompt Engineering Architecture**: Cách cấu trúc system prompts, few-shot prompting ở mức hệ thống (chuẩn hóa input/output).

## 3. Kỹ năng Biểu diễn Kiến trúc (Diagram & Visualization generation)
- **Tạo sơ đồ bằng Text (Text-to-Diagram)**: Khả năng sinh ra mã Mermaid, PlantUML hoặc Structurizr để vẽ sơ đồ.
  - Sơ đồ kiến trúc tổng thể (C4 Model hoặc Block Diagram).
  - Sơ đồ luồng dữ liệu (Sequence Diagram, Data Flow Diagram).
- **Thiết kế cấu trúc UI/UX (UI/UX Hierarchy Mapping)**: Ánh xạ từ dòng người dùng sang cấu trúc màn hình dạng cây (Tree structure) hoặc dạng JSON.

## 4. Kỹ năng Viết Phân tích Đặc tả Kỹ thuật (Technical Writing)
- Khả năng chuyển đổi yêu cầu kinh doanh (Business Requirements) thành ngữ cảnh kỹ thuật.
- Viết chi tiết đặc tả (Function Description) với đầy đủ Input/Output/Logic/Exceptions.
- Giữ văn phong chuyên nghiệp, rõ ràng, không mơ hồ (Unambiguous).
